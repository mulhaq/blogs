✨ WHAT THIS PROJECT DOES
════════════════════════════════════════════════════════════════════════════

✓ Streams LLM responses token-by-token from Groq API
✓ Uses Server-Sent Events (SSE) for real-time push to browser
✓ Shows tokens arriving in real-time (ChatGPT-like experience)
✓ Handles errors gracefully (timeouts, disconnects, API errors)
✓ Runs on a single machine (Java 21 + Maven)
✓ Includes production-ready code (blog-ready quality)
✓ Has zero external JavaScript dependencies


💻 TECHNOLOGY STACK
════════════════════════════════════════════════════════════════════════════

Backend:
  • Java 21 + Spring Boot 3.3
  • Spring AI (LLM integration)
  • Project Reactor (reactive streams)
  • Groq API (llama-3.3-70b-versatile model)

Frontend:
  • HTML 5 + CSS 3 + Vanilla JavaScript
  • ZERO external dependencies

Streaming:
  • Server-Sent Events (SSE) over HTTP
  • No WebSockets needed

📖 KEY CONCEPTS (If Curious)
════════════════════════════════════════════════════════════════════════════

Server-Sent Events (SSE)
  • HTTP's native streaming protocol
  • One-way server→client push
  • Simpler than WebSockets for this use case

Spring AI ChatClient
  • Abstraction over LLM providers
  • Works with Groq, OpenAI, Anthropic, etc.
  • Simplifies LLM integration

Virtual Threads (Java 21)
  • Ultra-lightweight threads
  • Can handle millions of concurrent connections
  • Much better than traditional thread pools
