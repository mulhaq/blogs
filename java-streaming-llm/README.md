# Streaming LLM with Groq & Server-Sent Events

A production-ready Spring Boot application demonstrating **real-time LLM streaming** using Groq API with **Server-Sent Events (SSE)**. Stream responses token-by-token from Groq's fast inference engine directly to your browser.

## 🎯 What It Does

This project showcases:

- **Real-time Token Streaming**: Each token arrives as it's generated by the LLM, providing a ChatGPT-like experience
- **Server-Sent Events (SSE)**: Uses HTTP streaming for true server-to-client push without WebSockets
- **Groq API Integration**: Leverages Groq's ultra-fast inference (70B parameter model)
- **Spring AI**: Seamless LLM integration with Spring Boot using OpenAI-compatible API abstraction
- **Vanilla Frontend**: Pure HTML/CSS/JS frontend (no build tools, no frameworks)
- **Error Handling**: Graceful timeouts, client disconnection handling, and error recovery
- **Production-Ready Code**: Clean, blog-ready code with proper logging and resource management

## 🔌 API Endpoint

### `POST /api/chat/stream`

**Request:**
```json
{
  "message": "Explain quantum computing in simple terms"
}
```

**Response:** Server-Sent Events stream

```
data: {"token": "Quantum"}
data: {"token": " computing"}
data: {"token": " is"}
data: {"token": " a"}
...
data: [DONE]
```

Each token arrives as the model generates it. The client parses these events and displays them with a typewriter effect.

## 📊 Key Concepts

### Server-Sent Events (SSE)

SSE is HTTP's native streaming protocol. Unlike WebSockets, it uses a single HTTP connection for server→client messages:

```javascript
// Client-side: parse SSE stream
const response = await fetch('/api/chat/stream', { method: 'POST', body: JSON.stringify(...) });
const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    const chunk = decoder.decode(value);
    // Parse 'data: ...' lines
}
```

**Advantages:**
- No extra dependencies (built into modern browsers)
- HTTP 1.1+ compatible
- Automatic reconnect support (optional)
- Simpler than WebSockets for one-way streaming

### Spring AI & ChatClient

Spring AI abstracts away LLM provider specifics. We configure it as OpenAI-compatible to use Groq:

```java
chatClient.prompt()
    .user(userMessage)
    .stream()           // Returns Flux<String> of tokens
    .content()
    .doOnNext(token -> ...)
    .doOnError(err -> ...)
    .doOnComplete(() -> ...)
    .blockLast();       // Wait for completion
```

### Backpressure & Timeout Handling

- **Timeout:** SSE emitter has a 30-second timeout. If no token arrives within 30s, the connection closes with an error.
- **Client Disconnect:** If the browser closes the connection, `doOnNext()` catches the `IOException` and completes gracefully.
- **Virtual Threads:** Uses `Executors.newVirtualThreadPerTaskExecutor()` for efficient async processing.

## 🐛 Troubleshooting

### `GROQ_API_KEY` Not Found
**Error:** `java.lang.IllegalStateException: API key not found`

**Solution:**
```bash
export GROQ_API_KEY="your-key-here"
mvn clean spring-boot:run
```

### Timeout After 30 Seconds
**Why:** SSE emitter has a 30-second default timeout. If Groq takes longer, the connection closes.

**Fix:** Increase timeout in `ChatController.java`:
```java
private static final long SSE_TIMEOUT_MS = 60000; // 60 seconds
```

### CORS Error in Browser
**Why:** The frontend at `http://localhost:3000` tries to call `http://localhost:8080/api/chat/stream`

**Solution:** Already handled! Check `@CrossOrigin(origins = "http://localhost:*")` in `ChatController.java`

### Compile Error: Java Version
**Error:** `[ERROR] Source option 1.8 is no longer supported. Use 1.9 or later.`

**Solution:** Ensure Java 21 is your `JAVA_HOME`:
```bash
java -version  # Should show Java 21
```
